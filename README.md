# LocalFounder-mock

単一HTML（`index.html`）で動く **MUI + React（UMD/CDN）** のプロダクトモック。ローカルで開くだけで試せます。  
**目的：** ベース全体像を先に作り、後から各機能を高速にブラッシュアップできる土台を提供すること。

## 概要

- **対象ユーザー：** 地域おこし協力隊（現役/OBOG）＋地方で起業したい個人（特に1,000万円以内の少額融資を目指す層）
- **価値仮説：**
	- 実験ログ → 6ヶ月PLへ自動反映し、学びを数値化
	- 資金調達ナビで締切/適合度を提示、申請タスクをStep-by-Stepで潰せる
	- 許認可トラッカーで地域差分を吸収し、抜け漏れを防止
- **ゴール：** 個人課金（Starter/Pro）＋定額パック（Loan Pack / Permit Concierge）の体験導線をPoCで検証

## 主な画面と機能

- ハッシュベースの簡易ルーティング（例：`#/dashboard`）

### Dashboard
- 任期残日数カード / 今週の3アクション（擬似AI）
- 実験サマリー、申請タイムライン、最近の学び要約

### Experiments（実験ログ）
- 追加・編集・削除（ダイアログ）／Table/Kanban表示切替
- 「AI学び要約（擬似）」：Drawerで要約・次アクション・タグを生成表示

### Plan（事業計画）
- BMC（9ブロック）と6ヶ月PLの編集・保存
- 「実験実績を反映」（ダミー計算でPLに反映）

### Funding（資金調達ナビ）
- 北海道×業種×段階でフィルタ
- 候補カードに適合スコア計算、申請タスクのStepper展開

### Permits（許認可トラッカー）
- 業種テンプレ展開（飲食/宿泊のサンプル）
- 所管・必要書類・費用・ETA・ステータス管理＋進捗バー

### KPI
- 月次KPIの入力と履歴テーブル

### 課金導線（ダミー）
- AppBar「アップグレード」→ Pricingダイアログ（Free / Starter / Pro）
- Fundingスコア<60でLoan Packバナー→チェックアウト風モーダル
- Permits未着手が2件以上でPermit Conciergeバナー

### Onboarding / Cohort（サブページ）
- 初期セットアップ／6週間プログラムの概要

## ダミーデータと保存方式

- 初回ロード時に以下をlocalStorageへSeed：
	- `startupProfile`（任期・業種・目標）
	- `experiments`（3件）
	- `fundingPrograms`（5件）
	- `permitTemplates`（飲食/宿泊）
	- `kpiRecords`（直近3ヶ月）
- 以後は操作に応じて差分保存（ページ更新しても保持）

## 技術スタック

- React 18 / ReactDOM 18（UMD / CDN）
- MUI v5（@mui/material）＋ Emotion（UMD）
- MUI Icons（UMD）
- 単一HTML：ビルドなし、JSは `<script>` に直書き
- 簡易Hash Router と Context、`useLocalStorageState` カスタムフック

## 動かし方

1. リポジトリ/ファイル一式の取得（`index.html` と `README.md` など）
2. ダブルクリックで `index.html` を開く（もしくはローカルHTTPサーバーで配信）
3. 画面左のナビまたはURLハッシュでページ遷移
4. CORSを避けたい場合は `python -m http.server 8080` 等でローカル配信してください（任意）

## 操作ガイド（クイック）

- **実験を追加：** Experiments → 「+ 実験を追加」
- **AI学び要約（擬似）：** 行メニュー→「AI要約」
- **PL編集：** Plan → 6ヶ月PLテーブル編集 → 自動計算
- **資金スコア：** Funding → 各カードの「適合スコアを計算」
- **申請タスク：** Funding → 「申請タスクを展開」
- **許認可進捗：** Permits → 業種選択→チェック更新
- **アップグレード：** 右上「アップグレード」→プラン選択（ダミー）

---

## 既知の制約

- すべてクライアントサイドのみ。サーバ連携・認証なし
- スコアリング・AI要約は擬似ロジック
- データはブラウザのlocalStorage依存（ブラウザを跨ぐと共有されません）
- Chartは簡易表示（数値/擬似グラフ）

---

## PoCのDoD（受け入れ条件）

- 単一ファイル `index.html` で、以下が全て操作可能
	- 主要ナビの遷移（Dashboard/Experiments/Plan/Funding/Permits/KPI/Settings）
	- 実験の追加/編集/削除、AI要約Drawer表示
	- BMC/PLの編集と保存、実験→PLのダミー反映
	- Fundingのスコア計算と申請タスクStepper
	- Permitsのテンプレ展開と進捗更新
	- Pricing/Loan Pack/Permit Concierge のモーダル表示
	- レスポンシブ動作（xs:1列 / md:2–3列目安）

---

## 開発方針・拡張計画

- **優先順位（P1→P3）**
	- **P1:** 実験→PL連動ロジックの現実化 / Fundingスコア規則の精緻化 / 許認可テンプレの地域差分UI
	- **P2:** 申請書ドラフト雛形（項目マッピング）/ 監査レポートPDF（クライアント生成）
	- **P3:** 軽量バックエンド（認証・保存API）/ 決済連携 / メンター・士業マーケット

- **タスク例**
	- 実験→PL：売上/原価の紐付けルールをJSONで定義
	- Fundingスコア：締切・段階・直近KPI増加を実データに接続
	- 許認可：所管/書類/費用/ETAを地域マスタでオーバーライド
	- PDF出力：申請サマリー・稟議向け1枚資料
	- A/B：Loan Pack同時訴求の転換率計測フック

## トラブルシュート

- **画面が真っ白：** ブラウザコンソールでエラー確認（CDNの読み込み失敗など）
